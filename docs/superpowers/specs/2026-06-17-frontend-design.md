# Дизайн фронтенда «Запись на созвон»

Дата: 2026-06-17
Статус: утверждён

## Принцип

Фронтенд — отдельная часть приложения, общается с бэкендом **только** через API-контракт
(`spec/dist/openapi.yaml`). Во время разработки бэкенд заменяется mock-сервером Prism.

## Стек

- React 18 + Vite + TypeScript
- Tailwind CSS + shadcn/ui (Radix-примитивы)
- React Router — маршрутизация
- TanStack Query — загрузка данных и кэш
- openapi-typescript + openapi-fetch — типы и типобезопасный клиент из контракта
- lucide-react — иконки (без эмодзи)
- Prism — mock API из `openapi.yaml` (порт 4010)

Каталог: `frontend/`.

## Визуальное направление (из ui-ux-pro-max)

- Стиль: минимализм, одна колонка, много воздуха, микро-взаимодействия.
- Шрифт: Inter (заголовки и текст).
- Палитра: primary `#2563EB`, accent `#059669`, фон `#F8FAFC`, текст `#0F172A`.
- Тема: **светлая / тёмная / системная** с переключателем (по умолчанию — системная).
  Реализация — собственный ThemeProvider (class-стратегия Tailwind `darkMode: "class"`),
  выбор сохраняется в localStorage, системная тема — через `prefers-color-scheme`.
- Избегать: сложные тени, 3D-эффекты.

## Маршруты и страницы

### Гость (публичный контур)
- `/` — шапка с профилем владельца (`GET /owner`) + карточки видов брони (`GET /event-types`).
- `/book/:eventTypeId` — детали типа (`GET /event-types/{id}`), выбор слота из 14-дневного
  окна (`GET /event-types/{id}/slots`): селектор дня + сетка времени; форма (имя / email /
  комментарий) → `POST /bookings`; экран подтверждения при успехе.

### Владелец (админский контур, без авторизации — отдельные маршруты)
- `/admin` — список предстоящих встреч (`GET /bookings`), сортировка по `start`.
- `/admin/event-types` — создание типа события (`POST /event-types`) + список существующих.

## Слой данных

- `src/api/schema.d.ts` — типы, сгенерированные из `openapi.yaml`.
- `src/api/client.ts` — openapi-fetch клиент; базовый URL из `VITE_API_BASE_URL`
  (по умолчанию `http://localhost:4010` — Prism).
- `src/api/hooks.ts` — хуки TanStack Query на каждый эндпоинт.

## UX-состояния

- Загрузка — skeleton.
- Пусто — empty-state («Нет свободных слотов», «Пока нет встреч»).
- Ошибка — сообщение + retry.
- Форма — валидация (required, email), ошибка под полем.
- `409 slot_taken` → тост «Слот уже занят» + перезапрос слотов.
- Время — табличными цифрами; локализованный формат.

## YAGNI (сознательно не делаем на этом шаге)

- Аутентификацию админки.
- Редактирование/удаление типов и броней.
- i18n (интерфейс на русском, без переключения языка).

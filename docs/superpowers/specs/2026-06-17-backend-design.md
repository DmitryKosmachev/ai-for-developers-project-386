# Дизайн бэкенда «Запись на звонок»

Дата: 2026-06-17
Статус: утверждён

## Принцип

Реализация идёт **от контракта** (`spec/dist/openapi.yaml`), а не от фреймворка.
Бэкенд — отдельный API для фронтенд-клиента. Бизнес-правила бронирования живут на
сервере. Хранилище — в памяти (сбрасывается при перезапуске).

## Стек

- Python 3.12 + FastAPI + Uvicorn
- Pydantic v2 — схемы по контракту
- pytest + httpx TestClient — тесты
- Хранилище — структуры в памяти

Каталог: `backend/`.

## Структура

```
backend/
  requirements.txt
  app/
    main.py            # FastAPI, CORS, обработчики ошибок -> {code, message}
    config.py          # SLOT_MINUTES=30, WINDOW_DAYS=14, WORK_START=9, WORK_END=18, OWNER_TZ
    models.py          # Owner, EventType(+Create), Slot, Booking(+Create), ApiError
    store.py           # in-memory репозиторий + сид (владелец + типы событий)
    scheduling.py      # чистая бизнес-логика: генерация слотов, проверки
    routers/
      owner.py
      event_types.py
      bookings.py
  tests/
    test_event_types.py
    test_slots.py
    test_bookings.py
```

## Бизнес-правила (scheduling.py — чистые функции)

- Сетка 30 минут; окно 14 дней от текущего момента; дни пн–пт; часы 09:00–18:00
  по `owner.timezone` (`Europe/Moscow`).
- Слот для типа длительностью N минут: начинается на 30-мин сетке, помещается до 18:00,
  не в прошлом, и не пересекается ни с одной существующей бронью (глобально, через
  пересечение интервалов). Пересечение интервалов автоматически закрывает требования
  «N смежных свободных ячеек» и «занятость общая для всех типов».
- `Booking.end = start + durationMinutes` — считает сервер.

## Эндпоинты (по контракту)

| Метод | Путь | Ответы |
|---|---|---|
| GET | /owner | 200 Owner |
| GET | /event-types | 200 EventType[] |
| GET | /event-types/{id} | 200 · 404 |
| GET | /event-types/{id}/slots?from&days | 200 Slot[] · 404 |
| POST | /event-types | 201 · 422 |
| POST | /bookings | 201 · 404 · 409 · 422 |
| GET | /bookings | 200 Booking[] (сортировка по start) |

## Ошибки → формат контракта `{ code, message }`

- `404 not_found` — неизвестный eventTypeId.
- `409 slot_taken` — слот занят/пересекается (ключевое требование шага).
- `422 validation_error` — start не на сетке / вне рабочих часов / вне окна / в прошлом;
  durationMinutes не кратно 30 или ≤ 0; невалидный email.
- Дефолтный обработчик `RequestValidationError` FastAPI переопределяется, чтобы
  отдавать `{code:"validation_error", message}` вместо стандартного формата.

## Прочее

- CORS для фронта (`http://localhost:5173`).
- Время в ответах — UTC ISO 8601 (`...Z`).
- Сид при старте: владелец + «Знакомство» (30), «Консультация» (60), «Быстрый созвон» (30).

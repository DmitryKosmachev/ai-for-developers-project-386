# Дизайн API-контракта «Запись на созвон»

Дата: 2026-06-16
Статус: утверждён, реализован в `spec/main.tsp`

## Цель шага

Зафиксировать внешнее поведение системы и API-контракт (Design First). Контракт —
единый источник правды, по которому фронтенд и бэкенд реализуются независимо.
Инструмент: TypeSpec → OpenAPI 3.0.

## Роли

- **Владелец календаря** — единственный заранее заданный профиль. Без авторизации.
  Создаёт типы событий, смотрит список всех предстоящих встреч.
- **Гость** — без регистрации. Смотрит виды брони, выбирает свободный слот в окне
  14 дней, оформляет запись.

## Принятые решения

| Вопрос | Решение |
|---|---|
| Сетка слотов | Фиксированная 30-минутная сетка. Событие на N минут занимает N/30 смежных ячеек. |
| Рабочее окно | Пн–Пт, 09:00–18:00, по `owner.timezone`. |
| Окно записи | 14 дней от текущей даты. |
| Данные брони | Имя + email + необязательный комментарий. |
| Занятость | Глобальная: пересекающиеся по времени брони запрещены даже для разных типов событий. |
| Формат времени | ISO 8601 (`utcDateTime`). |
| `Slot` | Вычисляемый объект, в БД не хранится. |
| `Booking.end` | Считает сервер (start + durationMinutes), от клиента не принимается. |

## Доменные сущности

- **Owner** — `{ name, timezone }`, только чтение.
- **EventType** — `{ id, title, description, durationMinutes }`; `durationMinutes` ≥ 30 и кратно 30.
- **Slot** — `{ start, end }`, вычисляемый, не хранится.
- **Booking** — `{ id, eventTypeId, start, end, guestName, guestEmail, comment?, createdAt }`.
- **Ошибки** — единая модель `{ code, message }`; коды: `not_found` (404), `slot_taken` (409), `validation_error` (422).

## Эндпоинты

### Гостевой контур
- `GET /owner` → `Owner`
- `GET /event-types` → `EventType[]`
- `GET /event-types/{id}` → `EventType` · 404
- `GET /event-types/{id}/slots?from&days` → `Slot[]` · 404
- `POST /bookings` → `Booking` (201) · 404 · 409 · 422

### Контур владельца
- `POST /event-types` → `EventType` (201) · 422
- `GET /bookings` → `Booking[]` (все предстоящие встречи, сортировка по `start`)

## Сознательно не включено (YAGNI)

- Редактирование/удаление типов событий.
- Отмена/перенос брони.
- Серверная фильтрация `/bookings` по типу/дате (фильтрует фронт).
- Пагинация (объёмы малы — 14 дней).
- Авторизация (нет в проекте).

Расширяется при необходимости: правим контракт → перегенерируем OpenAPI → синхронно
обновляем фронт и бэк.

## Ограничения контракта, проверяемые в реализации

Часть правил невыразима в OpenAPI и проверяется на бэкенде, а в TypeSpec
задокументирована в `@doc`:

- `durationMinutes` кратно 30 (в схеме только `minimum: 30`).
- `start` выровнен по 30-мин сетке, в рабочих часах и окне 14 дней.
- Помещаемость N смежных свободных ячеек для выбранного типа.
- Глобальная проверка пересечения броней.
